# 빈 템플릿 가이드

## 목적
사용자가 템플릿을 보기 전에 순수하게 요구사항을 생각하도록 유도한다.

---

## Phase A: Blank Canvas

### 빈 요구사항 템플릿

사용자에게 아래 **빈 템플릿**을 제시하고 직접 작성하도록 요청한다.

```markdown
# 요구사항 초안

## 프로젝트명
[직접 작성]

## 비즈니스 목표
1. [목표를 직접 작성하세요]
2. [어떤 문제를 해결하나요?]

## 사용자 페르소나
- **주 사용자**: [누가 사용하나요?]
- **페르소나 설명**: [어떤 사람인가요?]

## 핵심 기능 (최대 5개)
1. [가장 중요한 기능]
2. [두 번째로 중요한 기능]
3. ...

## 기술적 제약
- [반드시 지켜야 할 제약]

## 일정
- [목표 일정]

## 하지 않을 것 (Won't Have)
- [이번에 제외할 기능]
```

---

## 작성 가이드

### 좋은 요구사항 작성법

| 항목 | 나쁜 예시 | 좋은 예시 |
|------|----------|----------|
| 목표 | "좋은 서비스 만들기" | "회원가입 전환율 30% 향상" |
| 사용자 | "사람들" | "20~30대 직장인, 모바일 우선" |
| 기능 | "회원 관리" | "이메일 인증 기반 회원가입" |
| 제약 | "빨리" | "2주 내 MVP 출시" |

### 구체화 질문 (모호할 때)

```
- "어떤 상황에서?" → 구체적 시나리오 유도
- "얼마나 자주?" → 빈도/트래픽 파악
- "누가?" → 사용자 명확화
- "왜?" → 비즈니스 목적 확인
```

---

## STOP Protocol

Phase A 완료 후 반드시 사용자 입력 대기:

```
---
👆 빈 템플릿을 작성해주세요.
작성 완료 후 "완료" 또는 "다음"이라고 입력해주세요.
```
