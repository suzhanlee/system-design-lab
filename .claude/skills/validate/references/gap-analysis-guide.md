# Gap 분석 가이드

## 목적
Phase C에서 사용자의 예측과 실제 검증 결과를 비교하여 학습 효과를 극대화한다.

---

## Gap Matrix 템플릿

```markdown
# Gap 분석 보고서

## 분석 일시
[날짜]

## Gap Matrix

| 항목 | 내 예측 | 실제 결과 | Gap | 학습 포인트 |
|------|---------|-----------|-----|-------------|
| 기술 스택 호환성 | ✅ PASS | ⚠️ WARN | 🔻 | [무엇을 놓쳤는가?] |
| 외부 의존성 | ✅ PASS | ✅ PASS | = | - |
| 예외 케이스 | ❌ FAIL | ❌ FAIL | = | 예측 정확! |
| ... | ... | ... | ... | ... |

## Gap 범례
- 🔻 = 예측보다 나쁨 (예측 실패)
- = = 예측 일치
- 🔺 = 예측보다 좋음 (과도한 비관)

## Gap 요약
- 예측 일치: N개
- 예측 실패 (비관적): M개
- 예측 실패 (낙관적): K개 ← **주의 필요**

## 주요 발견 사항

### 1. 내가 놓친 것
- [예측하지 못한 문제점]

### 2. 내가 과도하게 우려한 것
- [실제로는 문제 없었던 항목]

### 3. 개선 액션
- [다음 검증 시 적용할 방법]
```

---

## Gap 분석 질문

### 예측이 빗나간 경우

```
Q1: 왜 이 문제를 예측하지 못했는가?
- 정보 부족? → 더 많은 정보 수집 필요
- 경험 부족? → 유사 사례 학습
- 사고 방식? → Red Team Thinking 적용

Q2: 어떤 질문을 했어야 했는가?
- [구체적인 질문 작성]

Q3: 다음엔 어떻게 다르게 접근할 것인가?
- [개선 방안]
```

### 예측이 정확한 경우

```
Q1: 무엇이 정확한 예측을 가능하게 했는가?
- [성공 요인 분석]

Q2: 이 패턴을 다른 곳에도 적용할 수 있는가?
- [일반화 가능성]
```

## Gap 분석 예시

```markdown
# Gap 분석 보고서

## Gap Matrix

| 항목 | 내 예측 | 실제 결과 | Gap | 학습 포인트 |
|------|---------|-----------|-----|-------------|
| 기술 스택 호환성 | ✅ PASS | ⚠️ WARN | 🔻 | Spring Security 6.x 변경사항 간과 |
| 외부 의존성 | ✅ PASS | ✅ PASS | = | 외부 API 문서 충실히 검토함 |
| 예외 케이스 | ❌ FAIL | ❌ FAIL | = | 예측 정확 - 네트워크 타임아웃 누락 |
| 일정 | ✅ PASS | ❌ FAIL | 🔻 | MVP 범위 과소 평가 |

## 주요 발견 사항

### 1. 내가 놓친 것
- Spring Security 6.x의 API 변경 (filter chain 설정 방식)
- MVP에 포함된 "비밀번호 찾기" 기능의 복잡도

### 2. 개선 액션
- 프레임워크 버전업 시 Breaking Change 확인 루틴 추가
- 각 기능별 예상 공수를 1.5배로 산정하는 규칙 적용
```

## 학습 효과 극대화

### 패턴 기록

```
자주 놓치는 패턴을 기록하세요:

### 나의 취약점 TOP 3
1. [가장 자주 놓치는 유형]
2. [두 번째]
3. [세 번째]

### 나의 강점 TOP 3
1. [가장 잘 예측하는 유형]
2. [두 번째]
3. [세 번째]
```

## STOP Protocol (Phase C 완료)

```
---
👆 Gap 분석을 완료해주세요.

[Gap Matrix를 제시]

분석 완료 후 "완료" 또는 "다음"이라고 입력해주세요.
Phase D (Refinement)로 진행합니다.
```
