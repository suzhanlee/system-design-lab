# 예측표 작성 가이드

## 목적
Phase A에서 사용자가 검증 결과를 미리 예측하여 비판적 사고를 훈련한다.

---

## Predict Matrix 템플릿

### 기본 형식

```markdown
# 검증 예측표

## 예측 일시
[날짜]

## 검증 대상
- [요구사항 파일 경로]

## 예측 항목

### 1. Feasibility (실현 가능성)

| 항목 | 예측 결과 | 예측 근거 |
|------|-----------|-----------|
| 기술 스택 호환성 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [왜 그렇게 생각하는가?] |
| 외부 의존성 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| 일정/리소스 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |

### 2. Completeness (완전성)

| 항목 | 예측 결과 | 예측 근거 |
|------|-----------|-----------|
| 기능 명세 완전성 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| 예외 케이스 포함 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| 비기능 요구사항 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |

### 3. Consistency (일관성)

| 항목 | 예측 결과 | 예측 근거 |
|------|-----------|-----------|
| 용어 통일 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| 요구사항 충돌 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| 모호성 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |

### 4. Dependencies (의존성)

| 항목 | 예측 결과 | 예측 근거 |
|------|-----------|-----------|
| 외부 API | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| DB 의존성 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |
| 타 시스템 연동 | ✅ PASS / ⚠️ WARN / ❌ FAIL | [근거] |

## 종합 예측
- **최종 예측**: ✅ PASS / ⚠️ WARN / ❌ FAIL
- **예상 이슈**: [가장 문제가 될 것으로 예상하는 항목]
```

---

## 예측 작성 요령

### ✅ 좋은 예측

```markdown
| 항목 | 예측 결과 | 예측 근거 |
|------|-----------|-----------|
| 기술 스택 호환성 | ⚠️ WARN | Spring Boot 3.x + Java 17 조합은 검증되었으나, 신규 도입 예정인 Redis 클러스터링은 팀 경험 부족 |
| 예외 케이스 포함 | ❌ FAIL | 회원가입 실패 케이스(중복 이메일, 잘못된 형식)만 있고, 네트워크 오류/타임아웃 케이스 누락 |
```

### ❌ 나쁜 예측

```markdown
| 항목 | 예측 결과 | 예측 근거 |
|------|-----------|-----------|
| 기술 스택 호환성 | ✅ PASS | 잘 될 것 같음 |
| 예외 케이스 포함 | ✅ PASS | 충분해 보임 |
```

## 예측 팁

### Red Team Thinking 적용

```
"이 요구사항이 실패하려면?"

1. 기술적 실패 시나리오
   - 외부 API 응답 지연
   - DB 커넥션 풀 고갈
   - 메모리 누수

2. 비즈니스 실패 시나리오
   - 사용자가 예상과 다르게 행동
   - 트래픽이 예상치 초과
   - 법적/규제 이슈

3. 일정 실패 시나리오
   - 핵심 인력 이탈
   - 요구사항 변경
   - 기술 부채 누적
```

## STOP Protocol (Phase A 완료)

```
---
👆 Predict Matrix를 완성해주세요.
완료 후 "완료" 또는 "다음"이라고 입력해주세요.
그러면 AI가 실제 검증을 수행합니다.
```
