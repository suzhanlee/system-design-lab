# Happy Path 시나리오 작성

## 작성 가이드

| 섹션 | 작성 요령 |
|------|-----------|
| Given | 구체적인 데이터 상태, 테이블 형식 활용 |
| When | 단일 행동, 구체적인 요청 파라미터 |
| Then | 검증 가능한 결과, 상태 코드/응답 필드 명시 |

**중요**: "어떤 데이터", "특정 값" 등 모호한 표현 금지. 구체적인 데이터를 작성하세요.

---

## Feature 1: URL 단축

### Scenario: 정상적인 URL 단축 요청

**Given**
```
사용자가 존재한다.
```

**When** (행동)
```
사용자는 긴 url 을 제공한다.
```

**Then** (검증)
```
15자 이하의 url 이 반환된다.
and status code 200
```

---

## Feature 2: 리다이렉트

### Scenario: 정상적인 리다이렉트 요청

**Given** (사전 조건)
```
db에 긴 url - short url 이 매핑되어 있다.
```

**When** (행동)
```
사용자가 짧은 url을 누른다.
```

**Then** (검증)
```
긴 url로 라우팅되며 해당 url로 요청을 보내 결과를 가져온다.
and status code 302
```

---

## Feature 3: 통계 조회

### Scenario: 정상적인 통계 조회 요청

**Given** (사전 조건)
```
db에 긴 url - 짧은 url - 조회 수 가 매핑된 table의 data 존재한다.
```

**When** (행동)
```
관리자가 특정 짧은 url의 조회수를 조회한다.
```

**Then** (검증)
```
조회수가 반환된다.
and status code 200
```
