# 대규모 시스템 설계 학습 로드맵

> 가상면접 사례로 배우는 대규모 시스템 설계 기초 (Alex Xu) 기반

---

## 학습 순서 개요

```
1단계: 입문     → URL단축기, Pastebin
2단계: 중급     → 웹크롤러, 알림, 채팅
3단계: 상급     → 타임라인, 이메일, 쇼핑몰
4단계: 심화     → 유튜브, 드라이브, 검색, 지도
```

---

## 1단계: 입문 - 기본 패턴 익히기

가장 단순한 구조로 시스템 설계의 기본기를 다집니다.

| 순서 | 프로젝트 | 핵심 학습 포인트 |
|-----|---------|-----------------|
| 1 | [URL 단축기](./01-url-shortener) | 해시/ID 생성, 키-값 저장소, 캐시 기본 |
| 2 | [Pastebin](./02-pastebin) | CRUD 설계, 데이터 모델링, 만료 정책 |

### 학습 목표
- [ ] 시스템 설계 면접 4단계 프레임워크 이해
- [ ] 요구사항 분석 및 제약사항 정의
- [ ] 기본적인 API 설계
- [ ] 데이터베이스 스키마 설계

---

## 2단계: 중급 - 실시간 & 비동기

비동기 처리와 실시간 통신 패턴을 익힙니다.

| 순서 | 프로젝트 | 핵심 학습 포인트 |
|-----|---------|-----------------|
| 3 | [웹 크롤러](./03-web-crawler) | 메시지 큐, 스케줄링, 분산 작업 처리 |
| 4 | [뉴스 알림](./04-news-alert) | Push/Polling/Long-Polling, WebSocket |
| 5 | [채팅 시스템](./05-chat-system) | 실시간 양방향 통신, 연결 관리, 접속 상태 |

### 학습 목표
- [ ] 메시지 큐를 활용한 비동기 처리
- [ ] 실시간 통신 방식 비교 및 선택
- [ ] 분산 시스템에서의 작업 조율
- [ ] 연결 상태 관리 전략

---

## 3단계: 상급 - 읽기/쓰기 최적화 & 대규모

트래픽이 많을 때의 최적화 전략을 배웁니다.

| 순서 | 프로젝트 | 핵심 학습 포인트 |
|-----|---------|-----------------|
| 6 | [SNS 타임라인](./06-news-feed) | 팬아웃, 쓰기 vs 읽기 최적화 트레이드오프 |
| 7 | [이메일 서비스](./07-email-service) | 샤딩, 대용량 저장소, 검색 인덱싱 |
| 8 | [전자상거래](./08-ecommerce) | 트랜잭션, 재고 관리, 분산 락, 결제 시스템 |

### 학습 목표
- [ ] 읽기/쓰기 패턴 분석 및 최적화
- [ ] 데이터베이스 샤딩 전략
- [ ] 분산 트랜잭션 처리
- [ ] 캐싱 계층 설계

## 4단계: 심화 - 미디어 & 복잡한 도메인

미디어 처리와 복잡한 알고리즘이 필요한 시스템입니다.

| 순서 | 프로젝트 | 핵심 학습 포인트 |
|-----|---------|-----------------|
| 9 | [유튜브](./09-youtube) | CDN, 트랜스코딩, 적응형 스트리밍 |
| 10 | [구글 드라이브](./10-google-drive) | 동기화, 버전 관리, 충돌 해결 |
| 11 | [검색 엔진](./11-search-engine) | 역색인, 랭킹 알고리즘, 분산 검색 |
| 12 | [지도 서비스](./12-maps) | 지오해시, 경로 탐색, 타일 서버 |

### 학습 목표
- [ ] 대용량 미디어 처리 파이프라인
- [ ] 분산 파일 시스템 설계
- [ ] 검색 인덱스 구조 및 최적화
